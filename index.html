Концепт

Что такое x402

Открытый платёжный протокол для машин. Вместо редиректов и логинов — стандартный HTTP-ответ 402 Payment Required, несущий платёжные инструкции.

01

Платёж в протоколе

Клиент (агент) шлёт обычный запрос. Сервер отвечает 402 с ценой и адресом оплаты в заголовке PAYMENT-REQUIRED.

02

USDC на Base

Расчёты в стейблкоине, микросуммы, почти нулевая комиссия. Подходит для тысяч вызовов в минуту.

03

Без аккаунтов

Никаких API-ключей вручную, никаких договоров. Есть баланс — есть доступ. Нет баланса — сервер вежливо отказал.

04

Для агентов, не людей

x402 спроектирован так, чтобы им пользовался софт, а не вы. Агент сам решает: заплатить и получить данные, или нет.

Поток

Как агент платит через x402-wallet

x402-wallet — CLI-кошелёк для x402-платежей: хранит ключи и подписывает платежи от имени агента. Сам цикл выглядит так:

1

Агент делает запрос

Агент стучится к платному эндпоинту. Например, запрашивает свежие рыночные данные или вызов LLM.

2

Сервер отвечает 402 Payment Required

В заголовке PAYMENT-REQUIRED — цена в USDC и адрес для оплаты. Агент видит: «столько стоит этот вызов».

3

x402-wallet подписывает платёж

Кошелёк агента формирует платёжную авторизацию (USDC), и клиентский SDK прикладывает её к повторному запросу. Подпись берёт на себя протокол.

4

Сервер проверяет и отдаёт данные

Платёж валидируется через facilitator (settlement), и агент получает ответ. Всё за одну-две сетевые итерации, без редиректов.

// Node.js (x402 client SDK)
import { x402 } from "x402";
const res = await x402.fetch("https://api.example.com/data"); // платит сам, если пришёл 402

Факты по спецификации: сервер отвечает 402 Payment Required с инструкцией в заголовке PAYMENT-REQUIRED; клиент платит через SDK. Точный тип кошелька (x402-wallet, CDP-managed и др.) зависит от вашей реализации — см. docs.x402.org.

Практика

Установка и запуск

Минимальный путь, чтобы ваш агент начал платить через x402. Нужен Node.js (или Python) и немного USDC в сети Base.

1

Поставьте x402 SDK и кошелёк

Официальный Python SDK — x402 (PyPI). CLI-кошелёк для автономных платежей агентов — x402-wallet. Сид-фразу держите локально, не в облаке.

pip install x402          # Python SDK
npm i -g x402-wallet     # CLI-кошелёк для агентов
2

Пополните баланс USDC

Переведите немного USDC в сети Base на адрес агента. Этого хватит на тысячи микровызовов.

3

Подключите к агенту

В коде агента используйте x402-клиент: он сам поймёт 402, подпишет платёж кошельком и повторит запрос.

import { x402 } from "x402";
const data = await x402.fetch(url); // платит автоматически
4

Запуск

Запустите агента. Первый платный запрос пройдёт через 402 → подпись → данные. Готово.

Команды выше — рабочий минимум по официальным SDK (PyPI: x402, CLI: x402-wallet). Точные имена пакетов и флагов уточняйте в документации под вашу версию: docs.x402.org.

А если не собирать агента с нуля?

Идея x402 — в том, чтобы агенты были полезными и самодостаточными. Мы собрали одного такого: он круглосуточно сканирует рынок BTC/ETH, ищет дивергенции RSI и объёмные импульсы и присылает сигналы, по которым можно ставить грид-ботов.

Это не «гарантия прибыли» и не финсовет — просто рабочий пример агента, который делает рутину за вас. Решать, торговать или нет, всё равно вам.
